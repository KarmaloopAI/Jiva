Jiva

The Jiva Framework is a powerful AI-driven framework that allows for the creation and management of autonomous, goal-based agents utilizing open-source Language Learning Models (LLMs) on local machines. It offers optional support for integrating proprietary models like OpenAI, Mistral AI, and Anthropic. Jiva provides robust systems for asynchronous operations, memory management, ethical task evaluations, and seamless API integration through FastAPI. The framework is designed to support autonomous systems in performing complex tasks while ensuring ethical compliance and efficient resource utilization.

Overview

The Jiva project is an advanced autonomous agent framework implemented primarily using Python. It employs asynchronous programming heavily with a focus on APIs, task management, and integration with various language models (LLMs). The architecture is modular, with a clean separation of concerns across components such as API management, agent execution, memory handling, and ethical evaluations using a multi-container setup via Docker and Docker Compose.

Primary Components

Jiva Agent

Act as the core autonomous agent handling tasks, managing memory, and executing actions based on sensor inputs.

Design Patterns

Asynchronous Programming

To handle multiple concurrent operations efficiently, allowing the system to perform tasks like web scraping and API calls without blocking.

Found in: Implemented using `asyncio`, `aiohttp`, and integrated across the system for file operations, web interactions, and API requests.

Strategy Pattern

To select different language model providers at runtime based on configuration and needs.

Found in: Located in `llm_providers` where different LLM API providers like OpenAI, Anthropic, etc., are implemented.

Dependency Injection

For managing and injecting dependencies, especially LLM providers and configuration options.

Found in: Configuration files like `config.json` and example setup in various provider classes.

Observer Pattern

To react to sensor data inputs in real-time, allowing the agent to respond to changes in its environment.

Found in: In `core/sensor_manager.py`, where different sensors contribute data.

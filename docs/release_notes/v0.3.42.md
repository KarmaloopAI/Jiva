# Jiva v0.3.42 - Krutrim Reasoning Model Config Fixes

**Release Date:** March 23, 2026  
**Type:** Bug Fix Release

## Overview

This release fixes two bugs in the Krutrim reasoning model configuration within the HTTP session manager. Both issues affected users running Jiva with Krutrim-backed reasoning models.

## Changes

### Bug Fixes

#### Reasoning Model Name Fallback

- Fixed an issue where the reasoning model would fail to initialise if `model` was not explicitly set in the config
- The `model` field now falls back to `defaultModel` from the reasoning config when `model` is absent, matching the behaviour expected by flexible deployment configurations

#### `useHarmonyFormat` No Longer Hardcoded

- Fixed `useHarmonyFormat` being hardcoded to `true` for all Krutrim reasoning models
- Previously, the Harmony request format was unconditionally applied regardless of the model in use, which caused compatibility issues for Krutrim reasoning models that do not require it (e.g. models other than `gpt-oss-120b`)
- `useHarmonyFormat` now reads from the reasoning model config and defaults to `false` when not specified

**Affected file:** `src/interfaces/http/session-manager.ts`

## Upgrade Instructions

```bash
npm install -g jiva-core@0.3.42
```

## Notes

No configuration changes are required. If you were using a Krutrim reasoning model without an explicit `model` field, or experiencing unexpected request format errors, upgrading to v0.3.42 resolves both issues.
